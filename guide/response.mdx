---
title: "通用响应结构"
---

Phx Broker OpenAPI 采用统一的响应格式，以便于客户端进行一致性的错误处理和数据解析。

## 响应规范 (JSON)

所有请求均会返回一个包含 `code`, `message`, `data` 字段的 JSON 对象。

| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| `code` | `int` | 状态码。**200** 表示成功，非 200 表示失败。 |
| `message` | `string` | 结果描述。成功时通常为 "success"，失败时为错误原因。 |
| `data` | `object` | 业务数据。失败时可能为 `null`。 |

## 响应示例

### 1. 成功响应
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "uid": 1000000000001
  }
}
```

### 2. 失败响应 (如鉴权失败)
```json
{
  "code": 401,
  "message": "Invalid signature",
  "data": null
}
```

## 状态码速查

- **200**: 请求处理成功
- **400**: 参数校验错误
- **401**: 鉴权失败（API Key 错误、签名错误、时间戳过期）
- **404**: 资源不存在（用户不存在、券商不存在）
- **429**: 触发频率限制
- **500**: 系统内部错误
